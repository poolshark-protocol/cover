RESOLUTION:

We had an early return in `Positions.add()` when `params.amount` (i.e. the liquidity amount) was zero.

This was incorrect as it should have reverted.

It was creating a position with an epoch of 0, which could effectively claim at any tick in its range.

Since their liquidity would be 0, I don't think it is possible for this to have done any damage.

The moment they added a non-zero amount of liquidity, the epoch would have been set as state.accumEpoch and the position would have functioned as normal.
